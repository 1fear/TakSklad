## SkladBot rate limit

- SkladBot подтвердил лимиты: общий `/v1/*` - 60 запросов/мин, `/v1/requests/*`, `/v1/warehouse/*`, `/v1/report/stock`, `/v1/report/transactions` - 120 запросов/мин. При 429 API возвращает `Retry-After`.
- Со стороны TakSklad оставлена безопасная пауза `SKLADBOT_REQUEST_DELAY_SECONDS=2` на VDS.
- Backend `SkladBotClient` теперь выдерживает паузу перед каждым GET/POST и после 429 ждет полный `Retry-After`, вместо быстрого перебора токенов.
- Desktop fallback поднят с прежних `0.05` до безопасного дефолта `2.0` секунды и больше не зажимает сохраненную настройку вниз.

## Агрегационные КИЗы коробов

- Принято правило склада: один агрегационный КИЗ короба закрывает `50` блоков по любому Chapman SKU.
- Реализация без миграции БД: `scan_codes.raw_payload` хранит `scan_type` (`unit` или `aggregate_box`) и `block_quantity` (`1` или `50`).
- Backend при скане агрегата:
  - определяет SKU по префиксу кода;
  - проверяет совпадение SKU с текущей позицией;
  - запрещает скан, если в позиции осталось меньше `50` блоков;
  - увеличивает `order_items.scanned_blocks` на `50`;
  - duplicate/return/reset работают по самому агрегационному коду.
- Desktop продолжает использовать то же поле сканирования, но прогресс считает в блоках: обычный КИЗ `+1`, коробный агрегат `+50`.
- Клиентские КИЗ-отчеты выводят агрегированный код одной строкой с `Кол-во блок = 50`, без создания фиктивных внутренних КИЗов.
