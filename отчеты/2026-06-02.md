## Исправление Chrome `Не защищено` на taksklad.uz

- Подтверждено: сертификат Let's Encrypt для `taksklad.uz` действительный, DNS ведет на VDS `135.181.245.84`.
- Причина риска: frontend/API не были полностью закрыты security headers, а nginx за Traefik передавал backend внутренний `$scheme` вместо публичного `https`.
- Исправлено:
  - на frontend добавлены HSTS, CSP, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`;
  - CSP включает `upgrade-insecure-requests` и `block-all-mixed-content`;
  - `X-Forwarded-Proto` в nginx proxy зафиксирован как `https`;
  - Traefik security middleware добавлен на frontend, backend API и adminer-router.
- Деплой:
  - restore point: `/opt/taksklad/restore_points/pre-web-https-security-20260602T080353Z`;
  - пересобраны и пересозданы `frontend` и `backend-api`;
  - случайно поднятый `adminer` сразу остановлен и удален.
- Проверено:
  - `http://taksklad.uz/` делает `308` на HTTPS;
  - `https://taksklad.uz/`, `https://www.taksklad.uz/` и `https://api.taksklad.uz/health` отдают security headers;
  - frontend bundle не содержит hardcoded `http://taksklad.uz` или `http://api.taksklad.uz`;
  - серверный acceptance-smoke: backend health OK, compose running OK, Google/backend sync OK;
