## Исправление Chrome `Не защищено` на taksklad.uz

- Подтверждено: сертификат Let's Encrypt для `taksklad.uz` действительный, DNS ведет на VDS `135.181.245.84`.
- Причина риска: frontend/API не были полностью закрыты security headers, а nginx за Traefik передавал backend внутренний `$scheme` вместо публичного `https`.
- Исправлено:
  - на frontend добавлены HSTS, CSP, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`;
  - CSP включает `upgrade-insecure-requests` и `block-all-mixed-content`;
  - `X-Forwarded-Proto` в nginx proxy зафиксирован как `https`;
  - Traefik security middleware добавлен на frontend, backend API и adminer-router.
- Деплой:
  - restore point: `/opt/taksklad/restore_points/pre-web-https-security-20260602T080353Z`;
  - пересобраны и пересозданы `frontend` и `backend-api`;
  - случайно поднятый `adminer` сразу остановлен и удален.
- Проверено:
  - `http://taksklad.uz/` делает `308` на HTTPS;
  - `https://taksklad.uz/`, `https://www.taksklad.uz/` и `https://api.taksklad.uz/health` отдают security headers;
  - frontend bundle не содержит hardcoded `http://taksklad.uz` или `http://api.taksklad.uz`;
  - серверный acceptance-smoke: backend health OK, compose running OK, Google/backend sync OK;
  - общий `acceptance_status.sh` сейчас остается `failed` только по не связанным с HTTPS пунктам: 23 активных заказа без номера SkladBot и незакрытые ручные GO/NO-GO чекбоксы релиза;
  - локально `./.venv/bin/python -m unittest discover -s tests` - 247 tests OK;
  - `docker compose --env-file deploy/vds/.env.example -f deploy/vds/docker-compose.yml config` - OK;

## Исправление входа и HTTPS

- Проверено, что backend на VDS принимает рабочие данные входа через `https://taksklad.uz/api/v1/auth/login`.
- Найдена причина ошибки формы: frontend был собран с прямым URL `https://api.taksklad.uz` и мог обходить same-origin web-gate.
- Исправлено:
  - frontend больше не использует `VITE_TAKSKLAD_API_URL`;
  - frontend больше не читает старый `taksklad-web-config` из `localStorage`;
  - все web-запросы идут через `/api` на текущем домене;
  - добавлен HSTS-заголовок;
  - добавлен явный HTTP-router Traefik с permanent redirect на HTTPS.
- Доставлено на VDS и проверено:
  - новый JS bundle не содержит `https://api.taksklad.uz`;
  - `http://taksklad.uz/` возвращает `308 -> https://taksklad.uz/`;
  - `https://taksklad.uz/` возвращает `Strict-Transport-Security`;
  - login возвращает `200`;
  - API с cookie возвращает `200`;
  - API без cookie возвращает `401`;
  - `./deploy/vds/acceptance_status.sh` вернул общий `status=ok`.
- По Chrome `Не защищено`: сертификат на сервере валидный Let's Encrypt для `taksklad.uz` и `www.taksklad.uz`. Если у пользователя Chrome продолжает показывать `Не защищено`, причина почти наверняка в старом DNS/cache вкладки после смены IP.
